# SKILLS

## 사용한 스킬

### sites:sites-building

- 개념: 포트폴리오, 랜딩 페이지, 대시보드 같은 웹사이트를 구조화하고 구현하기 위한 Codex 웹 제작 스킬입니다.
- 사용 시점: 기존 React/Vite 포트폴리오의 화면 구조와 디자인을 변경하고, GitHub Pages에 배포 가능한 정적 웹사이트 형태를 유지해야 할 때 사용했습니다.
- 사용 환경: `my_portfolio` 폴더의 React + TypeScript + Vite + TailwindCSS 프로젝트에서 사용했습니다.
- 이번 작업 적용 내용: 참고 이미지의 다크 grid 배경, 큰 Hero 타이포, 제품형 카드, 대시보드 프리뷰, 중앙형 경력 타임라인을 현재 포트폴리오 구조에 맞게 반영했습니다.

## 관련 기술 스택

### React

- 개념: 화면을 컴포넌트 단위로 나누어 구성하는 프론트엔드 라이브러리입니다.
- 사용 시점: Hero, Featured, Projects, Career 같은 섹션을 재사용 가능한 컴포넌트로 분리할 때 사용합니다.
- 사용 환경: 브라우저에서 실행되는 포트폴리오 UI 구현에 사용했습니다.

### TypeScript

- 개념: JavaScript에 타입을 추가해 데이터 구조와 컴포넌트 사용을 더 안전하게 만드는 언어입니다.
- 사용 시점: 경력, 프로젝트, 스킬 데이터의 필수 필드와 형태를 명확히 관리할 때 사용합니다.
- 사용 환경: `src/types`와 `src/constants`의 포트폴리오 콘텐츠 모델링에 사용했습니다.

### Vite

- 개념: 빠른 개발 서버와 정적 빌드를 제공하는 프론트엔드 빌드 도구입니다.
- 사용 시점: 로컬 미리보기와 GitHub Pages 배포용 `dist` 빌드를 만들 때 사용합니다.
- 사용 환경: `npm run dev`, `npm run build` 명령으로 실행되는 프로젝트 환경입니다.

### TailwindCSS

- 개념: 미리 정의된 유틸리티 클래스로 레이아웃, 색상, 여백, 반응형 스타일을 빠르게 구성하는 CSS 프레임워크입니다.
- 사용 시점: 다크 테마, 카드, 타임라인, 버튼, grid 배경 같은 시각 요소를 일관되게 적용할 때 사용합니다.
- 사용 환경: `tailwind.config.js`의 디자인 토큰과 `src/styles/index.css`의 전역 스타일에 사용했습니다.
